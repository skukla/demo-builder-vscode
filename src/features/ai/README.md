# AI Feature

## Purpose

This feature has two responsibilities:

1. **AI setup verification** — health-checks the AI context files generated at project creation
2. **VS Code chat participant** — a `@demo-builder` participant for conversational project queries (Phase 2, not yet active)

The AI context files themselves are generated by three services in `src/features/project-creation/services/` and exposed to AI agents via the standalone MCP server (`src/mcp-server.ts`).

---

## Files

| File | Purpose |
|------|---------|
| `aiSetupVerifier.ts` | Checks for the four AI context file types and reports missing/malformed files |
| `vscodeChatParticipant.ts` | Registers the `@demo-builder` VS Code chat participant (Phase 2 — exported but not yet wired into `extension.ts`) |
| `index.ts` | Public API exports |

---

## Key APIs

### `verifyAiSetup(projectPath, extensionDistPath): Promise<AiVerificationResult>`

Runs four checks:

| Check | What it looks for |
|-------|-------------------|
| `claudeMd` | `.claude/CLAUDE.md` exists and is non-empty |
| `mcpConfig` | `.claude/mcp.json` exists and is valid JSON with a `mcpServers` key |
| `mcpBinary` | `{extensionDistPath}/mcp-server.js` exists |
| `skillFiles` | At least one `.md` file exists in `.claude/skills/` |

Returns `AiVerificationResult` with a status per check and an overall `allPassed` boolean.

### `registerChatParticipant(): void`

Registers the `@demo-builder` VS Code chat participant. **Not yet called from `extension.ts`** — Phase 2 activation only. Exported so wiring it up requires a single import change.

## Architecture

The AI context files live in the project directory, not in this feature:

```
<projectPath>/
├── .claude/
│   ├── CLAUDE.md          ← generated by aiContextWriter.ts
│   ├── mcp.json           ← generated by mcpConfigWriter.ts
│   ├── settings.json      ← generated by mcpConfigWriter.ts (PostToolUse hook)
│   └── skills/
│       ├── sync-changes.md
│       ├── add-component.md
│       └── ...            ← generated by skillsWriter.ts
└── .mcp.json              ← generated by mcpConfigWriter.ts (Claude Code project scope)
```

The three writers live in `src/features/project-creation/services/`:
- `aiContextWriter.ts` — generates `.claude/CLAUDE.md`
- `skillsWriter.ts` — writes `.claude/skills/*.md` guides
- `mcpConfigWriter.ts` — generates `.mcp.json`, `.claude/mcp.json`, `.claude/settings.json`

The orchestrator is `projectFinalizationService.generateAIContextFiles()`, called as Phase 6 of project creation and from `configureHandlers.ts` for on-demand regeneration.

The **MCP server** is compiled from `src/mcp-server.ts` (not from inside this feature directory) to `dist/mcp-server.js`. It runs as a standalone stdio process — no VS Code dependency — and exposes six tools to AI agents:

| Tool | Description |
|------|-------------|
| `get_project` | Reads `.demo-builder.json` |
| `get_component_config` | Reads `.demo-builder.json` or a `.env` file |
| `update_project_config` | Writes `.demo-builder.json` or a `.env` file |
| `sync_storefront` | Git add / commit / push in the storefront directory |
| `list_blocks` | Lists block directory names |
| `get_block_source` | Reads all source files for a named block |

---

## Integration Points

- **Configure screen** (`src/features/dashboard/ui/configure/`): The AI Setup tab calls `verifyAiSetup()` to display health status and offers regeneration via `configureHandlers.ts`.
- **Project creation** (`projectFinalizationService.ts`): `generateAIContextFiles()` runs all three writers after project setup completes.
- **Extension entry point** (`extension.ts`): When Phase 2 ships, add `registerChatParticipant()` here.
